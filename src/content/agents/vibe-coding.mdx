## 真正的核心

Vibe Coding 的价值不是减少思考，而是把人的注意力从机械编码转移到任务定义、架构判断、结果验证与迭代决策。

## 一个可靠的工作流

先让 Agent 阅读项目结构，再限制本轮修改范围；每轮只解决一个可验证的问题；修改完成后必须运行测试、检查差异并记录失败原因。

## 最容易失败的地方

上下文过长会让 Agent 忽略关键约束；需求含糊会导致它用看似合理的实现填补空白；没有测试时，代码“能运行”并不等于行为正确。

## 可复用原则

- 先让 Agent 解释现状，再让它修改。
- 把验收标准写成可以执行的检查。
- 大改动拆成多个小提交。
- 始终保留人工审查与回滚能力。
